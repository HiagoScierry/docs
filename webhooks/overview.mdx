Webhooks são requisições HTTP `POST` enviadas automaticamente ao seu servidor quando eventos acontecem na plataforma. Configure em **Plataforma → Integrações → Webhooks**.

## Como funciona

```
Evento ocorre na Cordialy
  → Cordialy enfileira a entrega
  → POST para seu endpoint com payload JSON + assinatura
  → Seu servidor processa e responde 2xx
```

Sem necessidade de polling — você recebe os dados em tempo real.

## Estrutura do envelope

Todo evento segue a mesma estrutura:

```json
{
  "event_type": "lead.created",
  "store_id": "uuid-da-loja",
  "occurred_at": "2026-06-20T14:30:00.000Z",
  "payload": { ... }
}
```

| Campo | Tipo | Descrição |
|---|---|---|
| `event_type` | string | Nome do evento |
| `store_id` | uuid | ID da loja que gerou o evento |
| `occurred_at` | ISO 8601 | Momento em que o evento ocorreu |
| `payload` | object | Dados específicos do evento (ver [Eventos & Payloads](/webhooks/events)) |

## Eventos disponíveis

| Evento | Trigger |
|---|---|
| `lead.created` | Novo lead criado via WhatsApp ou API |
| `lead.status_changed` | Status do lead alterado |
| `message.received` | Lead enviou mensagem via WhatsApp |
| `message.sent` | IA ou consultor enviou mensagem |
| `session.started` | Nova sessão de atendimento aberta |
| `session.ended` | Sessão de atendimento encerrada |
| `followup.sent` | Follow-up automático disparado |
| `appointment.created` | Agendamento criado para o lead |
